Addition 4: Frontend — Campaign Creation Flow
Add new frontend route /creator:

/creator/apply — Creator Program application form

Fields: legal name, organization name, country, city, website or social link, email, past event history (text), expected use of DaddyX (text)
Calls applyAsCreator() on CreatorRegistry
Status tracker: "Application under review" / "Approved" / "Suspended"
Note: approval is done by internal TicketDaddy team via admin panel
/admin/creators — Admin panel (TicketDaddy team only)

Pending queue: list of applicants with application details
"Approve" button → calls approveCreator(address) on CreatorRegistry
"Suspend" button
History of approved creators
/creator/dashboard — Approved creator dashboard

List of their campaigns: name, status, capital raised, token count, milestone status
"Create New Campaign" CTA
/creator/campaign/new — Campaign creation form (3 sections)

Section 1 — Event Details

Event name
Event date and time
Venue name + city + country
Event type (Concert / Festival / Sports / Conference / Comedy / Other)
Expected attendance
Total event budget (in USD)
Short description (160 chars — shown to fans)
Long description / pitch (markdown supported)
Upload: budget breakdown PDF, venue confirmation letter (stored on IPFS)
Section 2 — DaddyX Parameters (with live preview)

Number of tokens [N]: slider 10–1000, default 100
Initial token price [I]: number input (USD), default $5
Step factor [S]: slider 1.1x to 3.0x, default 1.5x (show: "each purchase makes token X% more expensive")
Payout factor [P]: slider 1.05x to S-0.01x, default 1.2x (show: "outbid holders guaranteed X% ROI")
Revenue share %: slider 5%–40%, default 20% (show: "token holders share X% of net event revenue")
Campaign end date: date picker (must be before event date)
Remedy plan: dropdown (Cancel + Full Refund / Postpone + Transfer / Partial Refund) + text field
Section 3 — Live ROI Preview (computed client-side, updates as parameters change)
Show a card:

"If all [N] tokens are sold through 5 purchase rounds, organizer raises: $X"
Animated Recharts price curve showing token price across 10 rounds
"Average early backer ROI (outbid at round 5): X%"
"Final holder revenue share per token if event achieves [Y]grossrevenue:[Y] gross revenue:[Y]grossrevenue:Z"
Revenue sensitivity table: 3 scenarios (50% of projection / 100% / 150%)
On submission → calls DaddyXFactory.createCampaign() with all parameters. Stores campaign details JSON to IPFS. Shows transaction pending → success with campaign URL.

Addition 5: Backend API Updates
POST /api/creator/apply — submit application to internal queue
GET /api/creator/status?wallet= — check creator application status
GET /api/admin/creators/pending — admin only: pending applications
POST /api/admin/creators/approve — admin only: trigger on-chain approval
POST /api/campaigns/create — record campaign after on-chain confirmation
POST /api/campaigns/[id]/milestone/request — organizer requests milestone release
POST /api/admin/campaigns/[id]/milestone/approve — admin approves milestone, triggers contract call

Add Creator and Milestone tables to Prisma schema:

prisma

model Creator {
  id          String   @id @default(cuid())
  wallet      String   @unique
  name        String
  country     String
  email       String
  status      String   @default("pending")
  appliedAt   DateTime @default(now())
  approvedAt  DateTime?
  campaigns   Campaign[]
}

model Campaign {
  id                String   @id @default(cuid())
  creator           Creator  @relation(fields: [creatorId], references: [id])
  creatorId         String
